# Skills_for_AI-Agents

Здесь собраны навыки и правила для ИИ-агентов, дабы они работали более эффективно, лучше понимали контекст и выполняли задачи в соответствии с ожиданиями.

Буду их постоянно обновлять, дорабатывать и добавлять новые. Если у вас есть идеи — пишите в Issues, а если есть новые навыки или доработки — присылайте Pull Request.

На данный момент есть следующие навыки:

- [ ] **AntiLoop** — навык, который предотвращает зацикливание ИИ-агента при работе с инструментами.
- [ ] **Code Review** — навык, который позволяет ИИ-агенту проводить полное ревью кода и давать рекомендации по улучшению.
- [ ] **Coder** — навык, позволяющий ИИ-агенту думать как продвинутый разработчик. Улучшает техническое мышление.
- [ ] **Planning** — планирование действий и декомпозиция большого запроса на множество мелких подзадач.
- [ ] **Skill-Maker** — позволяет ИИ-агенту создавать новые навыки.
- [ ] **High IQ** — навык, улучшающий абстрактное мышление и архитектурный анализ.

И есть 3 режима работы ИИ-агента:

- [ ] **Minimal Actions Mode** — минимальный режим, в котором ИИ-агент выполняет только необходимые действия для достижения цели и обдумывает свои действия, чтобы избежать лишних шагов. Позволяет сильно экономить токены и запросы к инструментам, но агент может не использовать все доступные инструменты и возможности.
- [ ] **Balanced Mode** — сбалансированный режим, в котором ИИ-агент использует доступные инструменты и возможности, но при этом старается избегать лишних действий. Позволяет немного экономить токены и запросы к инструментам. Можно использовать как основной режим работы ИИ-агента.
- [ ] **Extra Mode** — режим, в котором ИИ-агент использует все доступные инструменты и возможности ради достижения цели, при этом формируя план, обдумывая всё и приходя в итоге к полному результату. Жрёт очень много токенов и действий. Не советую для повседневного использования, только для особых задач. Может быть нестабильным.

## Стандарт скиллов

Каждый скийл имеет единую структуру:

```yaml
---
name: <имя-в-kebab-case>
description: <Описание на русском языке>
license: MIT
metadata:
  author: koda
  version: "1.0.0"
---
```

### Обязательные секции

1. **## Описание** — что делает скилл, какая у него цель.
2. **## Когда активировать** — триггеры и ситуации, когда следует использовать.
3. **## Философия** — основные принципы и подходы (### Принципы).
4. **## Обязательный цикл** — пошаговый порядок действий (Шаги или Фазы).
5. **## Чек-лист** — что проверить перед завершением.
6. **## Scope** — область применения, ограничения.

### Дополнительные секции

- **## Пример правильного поведения** — пример использования скилла.
- **## Ограничения** — когда скилл не применять, условия использования.
- **## Типичные примеры** — примеры применения в разных доменах.
