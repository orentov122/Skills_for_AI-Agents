# Balanced Mode (Сбалансированный режим)

## Описание
Режим баланса между глубоким анализом и эффективным выполнением. Сначала хорошо думаем, потом хорошо делаем. Режим по умолчанию, если не указан другой.

## Когда активировать
- Задачи средней сложности — не тривиальные, но и не мега-проекты
- Когда не указан конкретный режим

## Принципы
1. **Размышление.** Анализируем задачу, выявляем скрытые зависимости.
2. **План.** Формируем чёткий пошаговый план.
3. **Выполнение.** Выполняем план с качеством, не торопясь.
4. **Проверка.** Сверяем результат с исходным запросом.
5. **Вывод.** Формулируем, что сделано и почему.

## Цикл

### Шаг 1: Анализ задачи
1. Что именно просит пользователь? (цитируем/парафразируем)
2. Какие есть скрытые требования или подтексты?
3. Какие файлы/компоненты затронуты?
4. Какие зависимости между компонентами?
5. Какие есть альтернативные подходы?
6. Какой подход выбираем и почему?

### Шаг 2: План действий
```
План:
1. [Действие 1] — цель, ожидаемый результат
2. [Действие 2] — цель, ожидаемый результат
3. [Действие 3] — цель, ожидаемый результат
```

### Шаг 3: Выполнение с осознанностью
- Перед каждым вызовом: «Делаю X, чтобы получить Y»
- После каждого вызова: «Получил Z, это соответствует плану»
- Если результат не соответствует — останавливаемся и переосмысливаем

### Шаг 4: Проверка результата
1. Сверяем результат с запросом
2. Каждый пункт — done или pending
3. Если что-то не выполнено — доделываем

## Пример
```
Задача: «Добавь обработку ошибок в parse_config»

Анализ:
  - parse_config читает JSON, нет try/except
  - Нужно: FileNotFoundError, JSONDecodeError
  - Зависимости: вызывается из main.py и worker.py

План:
  1. Прочитать parse_config
  2. Добавить try/except с конкретными исключениями
  3. Добавить логирование ошибок
  4. Проверить, что вызывающие функции не сломаются

Проверка:
  - FileNotFoundError done
  - JSONDecodeError done
  - Логирование done
  - Вызывающие функции не затронуты done
```

## Чек-лист
- [ ] Понял, что именно просил пользователь
- [ ] Составил план перед началом действий
- [ ] Каждое действие было осмысленным
- [ ] Проверял результат каждого шага
- [ ] Сверил финальный результат с запросом
- [ ] Сформулировал вывод

## Scope
Для тривиальных задач анализ сокращён до 3 пунктов. Проверка результата обязательна всегда.
