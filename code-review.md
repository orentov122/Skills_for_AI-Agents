# Code Review (Полный аудит кода)

## Описание
Полная проверка кода проекта на наличие проблем в логике, UI/UX, переводах, безопасности и производительности. Каждый файл читается полностью — от первой до последней строки без пропусков.

## Когда активировать
- «Проверь весь код», «Проверь бота», «Аудит кода»

## Принципы
1. **Полное покрытие.** Каждый файл — прочитан полностью.
2. **Без правок без подтверждения.** Только список проблем.
3. **Конкретика.** Файл, функция, строка и объяснение.
4. **Группировка.** Проблемы по категориям.

## Цикл

### Шаг 1: Определение scope
Определить, какие файлы/директории проверять.

### Шаг 2: Полное чтение файлов
Прочитать КАЖДЫЙ файл проекта полностью. Если файл большой — читать частями (по 1000 строк), но ни один блок не пропускать.

### Шаг 3: Проверка по категориям

#### ЛОГИКА
- Мёртвый и недостижимый код
- Состояния гонки и асинхронность
- Обработка ошибок (глобальные `except: pass`, потеря stack trace)
- Вычисления (ошибки округления, деление на ноль)
- Состояния и жизненный цикл (утечка ресурсов)
- Данные и консистентность
- Зависимости (отсутствие timeout'ов, retry logic)
- Безопасность (логическая — обход прав, инъекции)

#### UI/UX
- Тексты и сообщения (hardcoded строки)
- Навигация (мёртвые кнопки, отсутствие подтверждения)
- Состояния интерфейса (устаревшие данные, нет индикаторов загрузки)

#### ПЕРЕВОДЫ
- Консистентность (смешивание языков)
- Качество (неполные переводы)
- Специфичные проблемы (переменные в переводе)

#### БЕЗОПАСНОСТЬ
- Аутентификация и авторизация
- Данные (SQL-инъекции, XSS, path traversal)
- Конфигурация (секреты в коде)

#### ПРОИЗВОДИТЕЛЬНОСТЬ
- Алгоритмы (O(n²) где можно O(n), N+1 запросы)
- Ресурсы (отсутствие кеширования, частые подключения)
- Асинхронность (blocking calls в async-коде)

#### МИНОРНЫЕ
- Код-стайл, архитектура, тестирование, конфигурация, документация

### Шаг 4: Формирование отчёта
```
## ЛОГИКА

**1. [Краткое название]**
Описание проблемы.
Где находится (файл, функция, строка).
Почему это проблема.
Возможные последствия.
```

### Шаг 5: Ожидание подтверждения
Жду подтверждения по каждому пункту перед внесением правок.

## Чек-лист
- [ ] Прочитал КАЖДЫЙ файл проекта полностью
- [ ] Проверил каждый файл по всем категориям
- [ ] Каждая проблема указана с файлом, функцией и строкой
- [ ] Проблемы сгруппированы по категориям
- [ ] НЕ вносил правки без подтверждения

## Scope
Читать ВСЕ файлы проекта. Не трогать другие языки перевода. Если проект большой — читать частями, не пропускать ни один.
