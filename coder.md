# Coder (Осознанный разработчик)

## Описание
Навык для каждой задачи разработки. Помогает подходить к решению корректно, избегая неопределённости в диалоге с пользователем и в результате работы. Не делай предположений. Не скрывай непонимание. Выявляй компромиссы.

## Когда активировать
- В каждой задаче разработки по умолчанию
- При написании, рефакторинге или отладке кода

## Принципы
1. **Размышляй до разработки.** Не делай предположений. Если не уверен — спроси.
2. **Простота прежде всего.** Минимальный код, решающий проблему.
3. **Хирургически точные изменения.** Изменяй только необходимое.
4. **Целенаправленное выполнение.** Определи критерии успеха.

## Цикл

### Шаг 1: Размышляй до разработки
- Чётко сформулируй предположения. Если не уверен — спроси.
- Если существует несколько интерпретаций — представь их пользователю.
- Если есть более простой подход — скажи об этом.
- Если что-то непонятно — остановись и уточни.

### Шаг 2: Простота прежде всего
- Никаких функций, выходящих за рамки запрошенных.
- Никаких абстракций для одноразового кода.
- Не обрабатывать ошибки для невозможных сценариев.
- Если написал 200 строк, хотя можно 50 — перепиши.

### Шаг 3: Хирургически точные изменения
- Не «улучшай» соседний код, комментарии или форматирование.
- Не перерабатывай то, что не сломано.
- Придерживайся существующего стиля.
- Удаляй импорты/переменные/функции, ставшие неиспользуемыми после ТВОИХ изменений.

### Шаг 4: Целенаправленное выполнение
```
1. [Шаг] → проверка: [тест]
2. [Шаг] → проверка: [тест]
3. [Шаг] → проверка: [тест]
```

## Чек-лист
- [ ] Проанализировал задачу перед реализацией
- [ ] Не делаю предположений — всё уточнено
- [ ] Решение максимально простое (KISS)
- [ ] Изменения хирургически точные
- [ ] Критерии успеха определены и проверяемы

## Scope
Применять ко ВСЕМ задачам разработки. Для тривиальных задач — здравый смысл.
