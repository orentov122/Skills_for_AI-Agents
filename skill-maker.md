# Skill Maker (Создание навыков)

## Описание
Мета-навык для создания новых навыков агентов. Помогает определить намерения, написать черновик SKILL.md, протестировать и итерировать. Соответствует стандарту agentskills.io.

## Когда активировать
- Пользователь хочет создать новый навык
- Есть рабочий процесс, который нужно превратить в навык
- Нужен черновик, тесты или итерация существующего навыка

## Принципы
1. **Ясность.** Однозначные формулировки — ни двусмысленностей.
2. **Практичность.** Навык должен реально работать.
3. **Итеративность.** Первый черновик требует доработки.
4. **Соответствие стандарту.** Следуй спецификации agentskills.io.

## Цикл

### Шаг 1: Определение намерений
1. Что этот навык должен позволить делать?
2. Когда навык должен срабатывать? (фразы, контекст)
3. Какой ожидаемый формат вывода?
4. Нужны ли тест-кейсы?

### Шаг 2: Интервью и исследование
Проактивно спрашивай о пограничных случаях, форматах, примерах файлов и критериях успеха. Не пиши тесты, пока не прояснишь детали.

### Шаг 3: Написание SKILL.md
Документ — на русском языке.

**Frontmatter:**
- **`name`**: Идентификатор (kebab-case, 1-64 символов, совпадает с именем директории)
- **`description`**: Когда срабатывает и что делает (1-1024 символов, на русском)

**Проверка имени:**
```
✅ Валидно: code-review, sprint-planner, github-flow
❌ Невалидно: Code-Review (верхний регистр), -my-skill (дефис в начале), my--skill (двойной дефис)
```

### Шаг 4: Руководство по написанию

**Анатомия:**
```
.agents/skills/<skill-name>/
    └── SKILL.md
        ├── YAML frontmatter (name, description)
        └── Markdown-инструкции
```

**Паттерны:**
- Метаданные (имя + описание) — всегда в контексте (~100 слов)
- Тело SKILL.md — в контексте при срабатывании (<500 строк)

Используй повелительное наклонение. Объясняй «почему» — LLM лучше работают, когда понимают логику.

## Чек-лист
- [ ] Имя навыка соответствует директории (kebab-case)
- [ ] Описание понятное, конкретное, 1-1024 символа
- [ ] Описание на русском языке
- [ ] Frontmatter заполнен (name, description, license, metadata)
- [ ] Навык протестирован на нескольких промптах
- [ ] Пользователь доволен результатом

## Scope
Только когда пользователь явно просит создать навык. После создания предложи протестировать.
