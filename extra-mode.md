# Extra Mode (Режим максимальных возможностей)

## Описание
Режим для сложных, многокомпонентных проектов. Делаем всё необходимое, но каждое действие — результат глубокого осмысления. Не экономим вызовы, но и не тратим их бездумно.

## Когда активировать
- «Режим максимальных возможностей», «глубокий анализ», «полный подход»
- Проекты с множеством файлов, зависимостей и компонентов
- Архитектурные изменения и рефакторинг
- Задачи, требующие понимания всей кодовой базы

## Принципы
1. **Системное мышление.** Проект — система взаимосвязанных компонентов.
2. **Понимание до изменения.** Не меняем ничего, пока не поймём систему целиком.
3. **Постепенное внедрение.** Сложные изменения — поэтапно, с проверкой.
4. **Учёт зависимостей.** Каждое изменение влияет на другие компоненты.
5. **Документирование решений.** Почему сделали именно так.

## Цикл

### Фаза 1: Полное понимание проекта
1. **Карта проекта.** Структура директорий, назначение каждого файла.
2. **Зависимости.** Как компоненты связаны (импорты, вызовы, данные).
3. **Архитектура.** Паттерны, слои, границы модулей.
4. **Критические точки.** Где чаще баги? Что самое хрупкое?
5. **Контекст.** История изменений (git log).

### Фаза 2: Стратегическое планирование
1. **Scope изменений.** Что меняем и что НЕ трогаем.
2. **Последовательность.** Порядок внесения изменений.
3. **Точки проверки.** Где остановиться и проверить.
4. **Fallback-план.** Что делать, если не по плану.
5. **Критерии успеха.** Как поймём, что успешно.

### Фаза 3: Выполнение с осознанностью
- Перед каждым действием: «Делаю X, потому что Y, ожидаю Z»
- После каждого действия: «Получил Z, это ожидаемо»
- При unexpected поведении — остановка и анализ

### Фаза 4: Комплексная проверка
1. Каждая точка проверки из плана — верифицирована.
2. Зависимые компоненты — проверены.
3. То, что раньше работало — продолжает работать.
4. Результат сверен с запросом.

## Пример
```
Задача: «Добавь поддержку WebSocket в бота»

Фаза 1 — Понимание:
  - Бот на aiogram 3.x, основной файл main.py
  - WebSocket нужно интегрировать в основной цикл

Фаза 2 — Планирование:
  1. Создать WebSocket-сервер как отдельный модуль
  2. Интегрировать в основной цикл
  3. Добавить конфигурацию
  4. Добавить аутентификацию подключений
  5. Добавить логирование

Фаза 3 — Выполнение:
  [read_file main.py] [read_file config.py]
  [create_new_file websocket_server.py]
  [edit_file main.py] [edit_file config.py]

Фаза 4 — Проверка:
  - WebSocket-сервер работает done
  - Основной бот не сломан done
  - Конфигурация добавлена done
```

## Чек-лист
- [ ] Составил карту проекта
- [ ] Понял зависимости между компонентами
- [ ] Учёл историю изменений
- [ ] Сформировал стратегический план
- [ ] Каждая точка проверки верифицирована
- [ ] Результат сверен с запросом

## Scope
Фаза понимания проекта обязательна перед любыми изменениями. Комплексная проверка обязательна.
