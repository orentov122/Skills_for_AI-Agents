# Minimal Actions Mode (Режим экономии действий)

## Описание
Каждый вызов инструмента должен быть строго обоснован. Максимум размышлений до действий, минимум вызовов, максимальная эффективность каждого вызова. «Подумать 10 минут, действовать 1 минуту».

## Когда активировать
- «Экономь действия», «минимум вызовов», «думай прежде чем действовать»
- Большие кодовые базы (каждый вызов — дорогая операция)
- Ограниченное контекстное окно

## Принципы
1. **Один вызов — одна цель.** Не делать несколько мелких там, где один большой справится.
2. **Максимальная информативность.** `read_file` читает сразу весь файл.
3. **Параллельные вызовы.** Где возможно — несколько инструментов одновременно.
4. **Минимум повторных вызовов.** Тщательный анализ ДО вызова.
5. **Размышление вместо проб и ошибок.**

## Цикл

### Фаза 1: Глубокий анализ (ДО любых действий)
1. **Понимание задачи.** Что нужно? Какие критерии успеха?
2. **Карта проекта.** Какие файлы затронуты? Как связаны?
3. **Стратегия.** Какой один-два вызова дадут максимум информации?
4. **План действий.** Пошаговый план с ожидаемыми результатами.
5. **Критерии остановки.** Как поймём, что задача решена?

### Фаза 2: Минимальный набор вызовов
- **Чтение.** Только необходимые файлы. Большие — нужные секции.
- **Поиск.** Один `grep_search` вместо нескольких.
- **Редактирование.** Один `edit_file` — все изменения сразу.
- **Проверка.** Один финальный `read_file` вместо нескольких промежуточных.

### Фаза 3: Проверка результата
- Сверяем результат с запросом
- Каждый пункт — done или pending
- Если pending — доделываем (стремимся к последнему вызову)

## Пример
```
Задача: «Перенеси константы из main.py, config.py, utils.py в constants.py»

Фаза 1 — Анализ (без вызовов):
  - 3 файла с константами → constants.py
  - Стратегия: параллельно прочитать 4 файла → один edit_file

Фаза 2 — Минимальные вызовы:
  [read_file main.py] [read_file config.py] [read_file utils.py] [read_file constants.py]
  ← параллельно, 4 вызова

  [edit_file constants.py] ← один вызов, все изменения

Фаза 3 — Проверка:
  [read_file constants.py] ← один финальный

Итого: 6 вызовов вместо 12+
```

## Scope
Фаза глубокого анализа обязательна перед первым вызовом. Объединение вызовов — обязательный принцип.
