# AntiLoop — Навык предотвращения зацикливаний

## Описание

Навык для предотвращения зацикливаний при использовании инструментов и размышлении. Применяется автоматически при любой работе с файлами и инструментами записи.

## Когда активировать

- Автоматически при любом использовании `edit_file`, `run_terminal_command`
- При любой ошибке поиска или изменения
- При повторных неудачных попытках вызова инструментов

## Философия

> Один промах — не ошибка. Два промаха — повод остановиться. Три — сигнал к смене стратегии.

### Принципы AntiLoop

1. **Ограничение на повторения.** Один и тот же вызов — не более двух раз подряд.
2. **Перечитай перед повтором.** Каждый повтор обязан использовать новые данные.
3. **Меняй стратегию.** Если два подхода не сработали — пробуй третий.
4. **Останавливайся.** Три шага без прогресса — сигнал к остановке.

## Обязательный цикл

### Шаг 1: Первая попытка

Вызови инструмент с наилучшими аргументами.

### Шаг 2: Анализ ошибки

Если инструмент вернул ошибку — не повторяй тот же вызов.

1. Перечитай relevant-часть файла через `read_file`
2. Найди точное содержимое для `old_string` / `path` / `query`
3. Определи, почему предыдущая попытка не сработала

### Шаг 3: Корректировка

Вызови инструмент с **скорректированными** аргументами.

### Шаг 4: Смена стратегии (если нужно)

Если инструмент не сработал второй раз:

- **edit_file** не нашёл `old_string` → перечитай файл, найди точное совпадение, проверь отступы
- **grep_search** не нашёл паттерн → попробуй другой паттерн, используй `file_glob_search`
- **read_file** вернул меньше строк → смести `line_start`, читай дальше
- **run_terminal_command** вернул ошибку → проверь путь, права, синтаксис

### Шаг 5: Остановка

Если 3 шага подряд не дали прогресса:

- Прекрати циклические вызовы
- Верни пользователю краткий статус
- Предложи альтернативу или задай уточняющий вопрос

## Чек-лист

- [ ] Я перечитал файл и знаю точное содержимое
- [ ] Мои аргументы отличаются от предыдущих
- [ ] Я понимаю, почему предыдущая попытка не сработала
- [ ] Я знаю, что буду делать при следующей неудаче

## Пример правильного поведения

```
# Попытка 1: edit_file не нашёл old_string
→ ОШИБКА: не найдено

# Попытка 2: read_file → нашёл точное содержимое → edit_file с правильным old_string
→ УСПЕХ

# Если edit_file снова не сработал:
→ read_file с другими line_start → проверяю отступы → edit_file
→ Если и это не помогло → спрашиваю пользователя
```

## Пример неправильного поведения (зацикливание)

```
# Попытка 1: edit_file с неправильным old_string
→ ОШИБКА: не найдено

# Попытка 2: edit_file с ТЕМ ЖЕ old_string
→ ОШИБКА: не найдено

# Попытка 3: edit_file с ТЕМ ЖЕ old_string
→ ОШИБКА: не найдено

# ← Зацикливание! Нужно было перечитать файл.
```

## Scope

- Применять ко ВСЕМ задачам, где используются инструменты записи (edit_file, run_terminal_command)
- Работает автоматически — не нужно активировать вручную
- Правило трёх попыток — обязательное, без исключений
